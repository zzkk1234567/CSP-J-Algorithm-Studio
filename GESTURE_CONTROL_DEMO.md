# 🎮 手势控制演示指南

## 快速开始

### 方法1: 使用测试页面
1. 在浏览器中打开 `test-gesture-control.html`
2. 查看自动运行的测试结果
3. 在"手势测试区域"中滑动以测试手势识别

### 方法2: 使用主应用
1. 在浏览器中打开 `csp-j-learning-tool.html`
2. 切换到"互动练习"标签
3. 选择任意算法演示
4. 首次使用会看到手势提示覆盖层
5. 在可视化区域滑动以控制步进

## 📱 手势操作说明

### 向左滑动 👈
- **功能**: 前进到下一步
- **等同于**: 点击"步进"按钮
- **条件**: 只在暂停状态下有效

### 向右滑动 👉
- **功能**: 后退到上一步
- **等同于**: 撤销上一步操作
- **条件**: 只在暂停状态下有效，且有历史记录

## 🎯 使用场景

### 场景1: 学习算法步骤
```
1. 选择"冒泡排序"算法
2. 点击"开始演示"
3. 点击"暂停"
4. 向左滑动 → 查看下一步
5. 向右滑动 → 回顾上一步
6. 重复步骤4-5，仔细学习每一步
```

### 场景2: 对比不同步骤
```
1. 运行算法到某一步
2. 暂停
3. 向右滑动回到前几步
4. 观察数据变化
5. 向左滑动前进
6. 对比前后状态
```

### 场景3: 移动设备学习
```
1. 在手机或平板上打开应用
2. 自动启用移动端优化
3. 使用手势控制更加便捷
4. 无需精确点击小按钮
5. 单手操作更流畅
```

## 🔧 配置选项

### 启用/禁用移动端优化
1. 点击右上角的设置图标 ⚙️
2. 找到"移动端优化"选项
3. 选择模式：
   - **自动**: 自动检测设备并启用
   - **启用**: 强制启用移动端优化
   - **禁用**: 关闭移动端优化

### 重置手势提示
如果想再次查看手势提示：
1. 打开浏览器开发者工具 (F12)
2. 进入Console标签
3. 输入: `localStorage.removeItem('gesture-hint-shown')`
4. 刷新页面

## 💡 提示和技巧

### 最佳实践
- ✅ 在暂停状态下使用手势，避免干扰播放
- ✅ 滑动距离至少50px才会触发
- ✅ 水平滑动比垂直滑动更容易识别
- ✅ 在可视化区域滑动效果最好

### 常见问题

**Q: 为什么滑动没有反应？**
A: 确保：
1. 算法演示已暂停（不在播放状态）
2. 滑动距离足够（至少50px）
3. 是水平滑动而非垂直滑动
4. 移动端优化已启用

**Q: 如何知道手势被识别了？**
A: 
1. 步骤说明会更新（显示"⬅️ 后退一步"或前进提示）
2. 可视化内容会相应变化
3. 步骤计数器会更新

**Q: 可以在桌面浏览器测试吗？**
A: 可以！
1. 打开开发者工具 (F12)
2. 切换到设备模拟模式
3. 选择移动设备（如iPhone或Android）
4. 使用鼠标模拟触摸滑动

## 📊 技术细节

### 事件流程
```
用户滑动
    ↓
MobileOptimization检测触摸
    ↓
计算滑动距离和方向
    ↓
触发自定义事件 (mobile-swipe-left/right)
    ↓
AlgorithmVisualizer监听事件
    ↓
调用stepForward()或stepBackward()
    ↓
更新可视化和状态
```
