# GitHub 日常开发与版本管理指南

本指南将帮助您轻松管理代码版本，实现自动备份、防乱、可回溯。

## 🚀 核心工具：`sync_github.bat`

为了简化操作，我为您创建了一个“一键同步脚本”：`d:\CSPJ\sync_github.bat`。

### 1. 日常开发流程 (如何上传)
每当您完成一个功能、修复一个 Bug，或者只是想保存当前进度时：

1.  双击运行 **`sync_github.bat`**。
2.  脚本会列出有变动的文件。
3.  **输入描述**：脚本会提示您输入一段文字。
    *   *推荐格式*：`动词 + 对象`，例如 "修复了冒泡排序动画"、"添加了二分查找代码"、"优化了界面颜色"。
    *   *作用*：这就像给存档起名字，方便以后查找。
4.  按回车，脚本会自动完成上传。

---

## 🛡️ 如何防止混乱 (最佳实践)

版本控制的核心在于 **"小步快跑"**。

*   **不要**：写了一整周代码，改了十几个文件，最后才提交一次。这样如果出问题，很难排查。
*   **要**：改完一个功能（比如“修好了冒泡排序”），就运行一次 `sync_github.bat`。
    *   这样您的 GitHub 仓库里就会有一串清晰的足迹：
        *   `10:00` 初始化项目
        *   `10:30` 修复冒泡排序
        *   `11:00` 添加二分查找
        *   `11:30` 调整界面颜色

## ⏪ 如何“倒回” (时光机)

如果您改乱了代码，想回到过去：

### 情况 A：还没提交 (还没运行 sync_github.bat)
如果您只是在编辑器里改乱了，想放弃刚才的所有修改：
1.  在项目文件夹右键 -> "Git Bash Here" (或者在终端中)。
2.  输入：`git checkout .`
3.  **后果**：所有未提交的修改都会消失，文件恢复到上次提交的状态。

### 情况 B：已经提交了 (已经运行了 sync_github.bat)
如果您提交了错误的代码，想回退到以前的版本：
1.  打开 GitHub 网页，点击 **Commits** (提交记录)。
2.  找到您想回到的那个版本（比如 "修复冒泡排序" 那次）。
3.  复制那个版本的 **ID** (一串字符，如 `a1b2c3d`)。
4.  在本地终端输入：
    ```bash
    git reset --hard a1b2c3d
    git push -f
    ```
    *(注意：这会强制覆盖当前代码，请谨慎使用)*

## 🔄 如何在多台电脑间更新

如果您在学校电脑改了代码，回家想继续写：

1.  在家里电脑的项目目录下。
2.  右键 -> "Git Bash Here"。
3.  输入：`git pull`
4.  这会将 GitHub 上最新的代码下载到本地。

---

## 总结
*   **上传/保存**：双击 `sync_github.bat` -> 输入描述 -> 回车。
*   **下载/更新**：`git pull`。
*   **防乱秘籍**：改完一个小功能就提交一次，描述写清楚。
